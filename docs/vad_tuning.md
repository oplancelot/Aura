# Aura VAD 与状态机断句调优指南

在实时语音翻译系统（Aura）中，如何精准地将连绵不绝的音频流切分成合理的长短句，是降低翻译延迟、提高上下文连贯性的核心。

以下参数决定了系统在各种场景下的表现。如果你觉得当前切句太碎（1秒一断），或者反应太慢，请修改对应的代码参数。

## 1. 状态机切句参数 (`ChunkingConfig`)
参数位置：`core/src/vad/state_machine.rs` 中的 `ChunkingConfig::default()`

| 参数 | 当前值 | 作用说明 | 调优建议 |
| :--- | :--- | :--- | :--- |
| `silence_close_ms` | `400` (毫秒) | **停顿判定时间**。只要静音超过这个时间，立刻认为一句话说完了，强制切分一个 `[FINAL]` 块。 | 实测 LJSpeech 最大自然停顿 ~362ms，`400` 是平衡值。调大则句子更完整但延迟更高；调小则响应更快但句子可能被截断。 |
| `provisional_start_ms` | `2000` (毫秒) | **长句暂存触发**。如果某人滔滔不绝讲话超过了这个时间还没停顿，为了让界面尽早出现字幕，开始进入"持续吐出暂存块"模式。 | 如果你希望翻译出得更早（哪怕只说了半句话），调小至 `1000`；如果你宁愿等一整句也不想看到闪烁的中间结果，调大至 `3000`。 |
| `provisional_interval_ms` | `200` (毫秒) | **暂存块吐出间隔**。进入暂存模式后，每隔多少毫秒吐出一个 `[PROVISIONAL]` 块给 AI 刷新字幕。 | 建议保持 `200-300` 即可。太小会导致 API 速率超限，太大会导致屏幕字幕卡顿。 |
| `hard_cut_ms` | `5000` (毫秒) | **硬切分时间**。防止有人一直不喘气说话导致内存爆炸。 | 实测最大语音段 ~5.3 秒，`5000` 确保及时出 ASR 结果。调大则等待更久但语义更完整。 |
| `hard_cut_overlap_ms` | `2000` (毫秒) | **硬切分重叠**。到达硬切分时，把前一句末尾几秒的音频拼接到下一句开头，以保留上下文语义。 | 保持默认 `2000` 即可。 |

## 2. 神经网络灵敏度参数 (`SileroVad`)
参数位置：`core/src/vad/silero.rs` 中的常量

| 参数 | 当前值 | 作用说明 | 调优建议 |
| :--- | :--- | :--- | :--- |
| `THRESHOLD_ON` | `0.10` | **人声触发置信度**。概率超过它，进入"说话"状态。 | 当前值 `0.10` 是因为 VAD v4 在 16kHz 下输出概率偏低。如果漏掉了清晰人声可再降低；如果环境噪音导致误触发可调高到 `0.20`。 |
| `THRESHOLD_OFF` | `0.05` | **静音触发置信度**。概率低于它，进入"静音"状态（并开始计算停顿判定时间）。 | 必须小于 `THRESHOLD_ON`，作为磁滞区间防止状态频繁翻转。 |

## 总结最佳实践
- **想要翻译更连贯、一长句一起出**：调大 `silence_close_ms`。
- **想要翻译更灵敏、话音刚落就有**：调小 `silence_close_ms`。
- **环境噪音太大导致 VAD 一直在录音不切断**：同时调高 `THRESHOLD_ON` 和 `THRESHOLD_OFF`。
- **VAD 在说话/不说话之间频繁跳动**：加大 `THRESHOLD_ON` 和 `THRESHOLD_OFF` 之间的间距。
