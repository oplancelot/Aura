# Aura VAD 与状态机断句调优指南

在实时语音翻译系统（Aura）中，如何精准地将连绵不绝的音频流切分成合理的长短句，是降低翻译延迟、提高上下文连贯性的核心。

以下参数决定了系统在各种场景下的表现。如果你觉得当前切句太碎（1秒一断），或者反应太慢，请修改对应的代码参数。

## 1. 状态机切句参数 (`ChunkingConfig`)
参数位置：`core/src/vad/state_machine.rs` 中的 `ChunkingConfig::default()`

| 参数 | 默认值 | 作用说明 | 调优建议 |
| :--- | :--- | :--- | :--- |
| `silence_close_ms` | `1200` (毫秒) | **停顿判定时间**。只要静音超过这个时间，立刻认为一句话说完了，强制切分一个 `[FINAL]` 块。 | 如果说话人经常“大喘气”导致一句话被切成两半，可以调大到 `1200 - 1500`。如果你想要同传那种“只要一停顿立刻出字幕”的极致低延迟，可以调小到 `500`。 |
| `provisional_start_ms` | `2000` (毫秒) | **长句暂存触发**。如果某人滔滔不绝讲话超过了这个时间还没停顿，为了让界面尽早出现字幕，开始进入“持续吐出暂存块”模式。 | 如果你希望翻译出得更早（哪怕只说了半句话），调小至 `1000`；如果你宁愿等一整句也不想看到闪烁的中间结果，调大至 `3000`。 |
| `provisional_interval_ms` | `200` (毫秒) | **暂存块吐出间隔**。进入暂存模式后，每隔多少毫秒吐出一个 `[PROVISIONAL]` 块给 AI 刷新字幕。 | 建议保持 `200-300` 即可。太小会导致 API 速率超限，太大会导致屏幕字幕卡顿。 |
| `hard_cut_ms` | `28000` (毫秒) | **硬切分时间**。防止有人一直不喘气说话导致内存爆炸。 | 建议保持默认，不超过 `30000`（30秒）。 |
| `hard_cut_overlap_ms` | `2000` (毫秒) | **硬切分重叠**。到达硬切分时，把前一句末尾几秒的音频拼接到下一句开头，以保留上下文语义。 | 保持默认 `2000` 即可。 |

---

## 2. 神经网络灵敏度参数 (`SileroVad`)
参数位置：`core/src/vad/silero.rs` 中的常量

| 参数 | 默认值 | 作用说明 | 调优建议 |
| :--- | :--- | :--- | :--- |
| `THRESHOLD_ON` | `0.45` | **人声触发置信度**。概率超过它，进入“说话”状态。 | 默认 `0.45`，适合大多数内录场景。如果环境特别吺导致总误判为说话，请**调高**到 `0.6` 或 `0.7`；如果漏掉了很多清晰人声，可**调低**到 `0.3`。注意：VAD 使用双阈值迟滞（hysteresis），进入说话状态需要超过 `THRESHOLD_ON`，退出说话状态需要低于 `THRESHOLD_OFF`。 |
| `THRESHOLD_OFF` | `0.25` | **静音触发置信度**。概率低于它，进入“静音”状态（并开始计算上文提到的停顿判定时间）。 | 必须永远小于 `THRESHOLD_ON`，作为磁滞区间，防止频繁在说话和不说话之间横跳。建议保持与 `THRESHOLD_ON` 有 0.15-0.2 的间距。 |

## 总结最佳实践
- **想要翻译更连贯、一长句一起出**：调大 `silence_close_ms` (如 1500-2000)。
- **想要翻译更灵敏、话音刚落就有**：调小 `silence_close_ms` 到 800 或更小。
- **环境噪音太大导致 VAD 一直在录音不切断**：调高 `THRESHOLD_ON` (如 0.6) 和 `THRESHOLD_OFF` (如 0.4)。
- **VAD 在说话/不说话之间频繁跳动**：加大 `THRESHOLD_ON` 和 `THRESHOLD_OFF` 之间的间距（至少保持 0.15 差值）。
